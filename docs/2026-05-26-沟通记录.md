## 二、Python 3.8 兼容性改造

服务器 Python 版本为 3.8.9，不支持 Python 3.10+ 类型注解语法。

### 2.1 类型注解替换

|---|---|---|
| `X \| None` | `Optional[X]` | excel_io, cli, batch_engine, health_check, vendor_profiles |
| `list[X]` | `List[X]` | excel_io, cli, batch_engine, health_check, vendor_profiles, reporting |
| `dict[X, Y]` | `Dict[X, Y]` | health_check, vendor_profiles |
| `tuple[X, Y]` | `Tuple[X, Y]` | health_check, file_transfer |
| `set[str]` | `Set[str]` | logger |

所有文件均添加 `from typing import ...` 导入。

### 2.2 验证方式

通过 `ast.parse()` 逐文件解析，确认无语法错误。

## 五、开发环境配置

### 5.1 工作流

```
kscc 改代码 → 本地文件更新 → PyCharm 自动 SFTP 同步到服务器
                               ↓
               你说"更新github" → commit + push
```

- 日常运行不需要 GitHub，PyCharm SFTP 同步到服务器即可
- GitHub 作为版本备份，手动触发 push

### 5.2 服务器信息

- 系统：CentOS 7，glibc 2.17
- Python：3.8.9
- 虚拟环境管理：virtualenvwrapper
- 虚拟环境路径：`/home/virtual_path/switch-patcher`
- 远程解释器路径：`/home/virtual_path/switch-patcher/bin/python3.8`
- 服务器无法联网（pip 不可用）

### 5.3 PyCharm 远程开发配置

**Deployment（SFTP 同步）**：
- `Tools → Deployment → Configuration → + → SFTP`
- Mappings: 本地项目目录 → 服务器 /switch-patcher
- 自动上传：`Tools → Deployment → Options → Upload changed files: Always`（选 Always 才能检测 kscc 外部修改）

**远程解释器**：
- `File → Settings → Project → Python Interpreter → ⚙ → Add → SSH Interpreter`
- Interpreter: `/home/virtual_path/switch-patcher/bin/python3.8`

### 5.4 离线依赖安装

服务器无法联网，需本地下载 whl 包后上传安装：

```bash
# 本地下载纯Python包（跨平台）
pip download -r requirements.txt -d packages/ --only-binary=:all: --no-deps

# Linux C扩展包需单独下载 cp38 + manylinux_2_17_x86_64 版本：
# PyYAML、bcrypt、cffi、cryptography、PyNaCl
# 从 PyPI JSON API 获取真实下载链接：
curl -sL "https://pypi.org/pypi/包名/版本/json" | python -c "import sys,json; ..."

# 上传 packages/ 到服务器后离线安装
workon switch-patcher
pip install --no-index --find-links=packages/ -r requirements.txt
```

**注意事项**：
- `pip download` 从 Windows 下载 Linux whl 时，需指定 `--platform manylinux_2_17_x86_64 --python-version 38 --implementation cp38 --abi cp38`
- bcrypt/cryptography/PyNaCl 使用 `abi3` 标签（兼容 cp38+），cffi 使用 `cp38-cp38` 标签
- CentOS 7 glibc 2.17 兼容 `manylinux_2_17` 和 `manylinux2014`
