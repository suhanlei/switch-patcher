# 打补丁流程及SSH登录次数统计

## 完整6步流程

| 步骤 | 动作 | SSH登录次数 |
|------|------|------------|
| 1. check_scp | TCP探测 → SSH登录检查SCP/SFTP状态 | 1 |
| 2. open_scp | SSH登录执行SCP/SFTP使能命令 → 保存配置 | 1 |
| 3. recheck_scp | SSH登录再次确认SCP/SFTP已开启 | 1 |
| 4. upload | paramiko SFTP上传补丁文件 + SSH登录验证文件完整性 | 2（1个paramiko SFTP + 1个netmiko SSH） |
| 5. activate | SSH预检查+激活+后检查（同一连接） | 1 |
| 6. finalize | 汇总报告 | 0 |

**完整流程每台交换机登录6次**（5次netmiko SSH + 1次paramiko SFTP Transport）

## 断点续跑减少次数

- SCP已开启 → 跳过步骤2/3，减少2次（变为4次）
- `dry_run` 模式 → 步骤5只做预检查，减少0次（同一连接内完成）
- 已上传成功 → 跳过步骤4，减少2次（变为3次）
- 已激活成功 → 跳过步骤5，减少1次（变为5次）
- 补丁已激活 → 步骤5预检查检测到后跳过，减少0次（同一连接内完成）

## 关键设计点

- 步骤5的预检查、激活、后检查在同一SSH连接中完成：激活过程中设备持续回显进度，SSH不会因空闲超时断开
- 每步完成后立即回写Excel，数百台设备执行中可随时打开Excel查看进度
